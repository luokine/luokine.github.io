GitHub执行命令失败

错误代码

本地已经启动了VPN

网页可以打开github。说明命令行在拉取/推送代码时并没有使用vpn进行代理
说明VPN配置，并没有生效，需要单独给git配置代理

socks5

http(实测有效)

关闭VPN

  1. windows会在当前路径下: C:\Users\TZQ.gitconfig
  2. 这个gitconfig文件会影响正常的git访问
  3. 比如：当vpn关闭后，再执行公司内部的git pull、push等操作会报错：502
  4. 建议：当关闭vpn的时候把此配置打开
